# Cours 01 — C'est quoi Next.js ?

## Ce que tu sais déjà
Tu connais HTML, CSS et JavaScript. C'est super, parce que Next.js utilise tout ça.
Mais avant de comprendre Next.js, il faut comprendre React.

---

## C'est quoi React ?

Imagine que tu construis une page web avec des blocs LEGO.
Chaque bloc, c'est un **composant** : un bouton, un menu, une carte produit.

Avec du HTML classique, si tu veux afficher 100 produits, tu dois écrire 100 fois le même code.
Avec React, tu crées **un seul bloc "produit"**, et tu l'utilises 100 fois.

React, c'est une **bibliothèque JavaScript** créée par Facebook pour construire des interfaces web avec des blocs réutilisables.

---

## Et Next.js alors, c'est quoi ?

Next.js, c'est React... mais en mieux et en plus complet.

Imagine React comme un moteur de voiture.
Next.js, c'est la voiture complète : le moteur (React) + les roues + le volant + les phares.

Next.js ajoute des trucs très utiles que React n'a pas tout seul :

| Ce que Next.js apporte | Explication simple |
|---|---|
| Le routing automatique | Tu crées un fichier, ça crée une page web automatiquement |
| Le rendu côté serveur | La page est préparée avant d'arriver dans ton navigateur |
| L'optimisation des images | Tes images sont automatiquement compressées et rapides |
| Une API intégrée | Tu peux faire un mini-serveur sans installer quoi que ce soit de plus |

---

## Pourquoi apprendre Next.js plutôt que React tout seul ?

Parce que :
1. Les entreprises utilisent beaucoup Next.js aujourd'hui
2. Il est plus rapide à démarrer
3. Il gère plein de choses compliquées à ta place
4. Il se connecte très bien avec NestJS et Prisma (qu'on verra plus tard)

---

## Un mot compliqué à connaître : le "rendu"

Le **rendu**, c'est le moment où ton code devient une vraie page que tu peux voir.

Il y a deux endroits où ça peut se passer :
- **Dans le navigateur** (ton Chrome, Firefox...) → c'est ce que fait React tout seul
- **Sur le serveur** (un ordinateur qui envoie la page déjà faite) → c'est ce que Next.js peut faire

Pour l'instant, retiens juste que Next.js peut faire les deux.

---

## Résumé du cours 01

- React = des blocs LEGO pour construire des pages web
- Next.js = React + plein d'outils pratiques déjà intégrés
- Next.js crée les pages automatiquement à partir de fichiers
- C'est ce qu'utilisent les vraies entreprises aujourd'hui
